Утилита для конвертации описания типов для Телепата из
формата xml во внутренний формат телепата.
Использовать
xml2tls ИмяXMLФайла [ИмяВыходногоФайла]
При отсутствии ИмяВыходногоФайла оно формируется из имени
входного файла заменой расширения на tls.

При работе телепат загружает все файлы с расширением tls из каталога BIN.
При каждом изменении используемых компонент происходит повторное
считываение этих файлов.

Для примера приложен файл, описание двух типов из 1С++.

Структура описания типов в xml формате.
Многие параметры тегов должны удовлетворять правилам идентификаторов 1С,
те состоять только из букв, цифр, подчеркиваний.
Допускается использовать точку.
Такие параметры далее в тексте описания обозначаются "идентификатор"

Корневой элемент <Types>.
Внутри идут отдельные типы (<type>)
Параметры типа:
name 	- обязательный, идентификатор. Внутреннее имя типа, обязано быть уникальным.
alias 	- необязательный, идентификатор. Имя типа для показа пользователю.
		Может быть неуникальным.
base	- необязательный. Перечисленные через запятую имена базовых типов.
creatable	- необязательный. Перечисленные через запятую имена для использования
			в СоздатьОбъект.
components	- необязательный. Число, обозначающее принадлежность типа
			только к определенной компоненте. Номера компонент:
			1 - Торговля, 2 - Бухгалтерия, 4 - Расчет, 8 - УРБД.
abstract	- Необязательный. Флаг 0 или 1. Признак, что объектов такого типа
			не существует, и тип просто введен как базовый для нескольких типов.
			
В <type> могут следовать описания атрибутов типа (<attribs>), методов типа (<methods>),
и предопределенных процедур (<handlers>).

В <attribs> идут описания атрибутов (<attrib>)
Параметры атрибута:
name		- Обязательный, идентификатор. Русское название атрибута.
alias		- Необязательный, идентификатор. Английское название атрибута.
components	- см. components в <type>.
type		- Необязательный. Имя типа атрибута.
В самом теле тэга <attrib> идет описание атрибута.

В <methods> и <handlers> идут описания методов (<method>)
Параметры метода
name		- Обязательный, идентификатор. Русское название метода.
alias		- Необязательный, идентификатор. Английское название метода.
components	- см. components в type.
Внутри тега <method> идут:
текст 		- необязательное, описание метода
<params> 	- необязательное, описание параметров
<return>	- необязательное, для функций описание возвращаемого объекта

В <params> перечисляются параметры метода (<param>)
Для <param> указываются:
name	- Обязательное. Имя параметра.
type	- Необязательное. Имя типа параметра. Допускается указывать через
		запятую несколько типов. Допускается через ; указывать набор
		возвращаемых значений. (Например "0;1")
default - Необязательное. Значение параметра по умолчанию. При отсутствии
		default параметр метода считается обязательным.
В тэге <param> указывается текст - описание параметра.

В <return> указываются:
type - см. type для <param>
InExprOnly - флаг. При 1 считается, что вызов данной функции как метода
	смысла не имеет.
Текст - описание возвращаемого значения. Учтите, при его показе телепат
вставит перед ним "Возращает: ".
