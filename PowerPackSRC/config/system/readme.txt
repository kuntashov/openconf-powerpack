1. ОПИСАНИЕ ВКЛЮЧЕННЫХ В ПОСТАВКУ КОМПОНЕНТ
	
	Существующими на настоящий момент скриптами для OpenConf используются 
следующие COM-объекты, реализованные в виде *.dll или *.wsc (Windows Script Componets)
файлов:

1.1 Компоненты, реализованные в виде DLL-файлов

	SelectDialog.dll
	SelectValue.dll
		Реализуют различные варианты диалогов для выбора значений из списка
		Авторы: Дмитрий Реутов (IAm)
		
	svcsvc.dll
		Реализует ряд сервисных функций для использования в скриптах, в том числе
		различного вида диалоги для выбора значений (из списка, из дерева, из меню)
		Авторы: Александр Орефков
		
	dynwrap.dll
		Библиотека, реализующая возможность вызывать многие функции Win32 API из
		скриптов.
		Авторы: Guenter Born и другие
		
	WshExtra.dll
		Функции для работы с буфером обмена из скриптов и другие
		Авторы: http://www.windowsitpro.com/
	
	macrosenum.dll
		Реализует объект-коллекцию для перебора макросов заданного скрипта OpenConf 
		Авторы: Артур Аюханов (artbear)
	
	ArtWin.dll
		Функции для работы с окнами: поиск окна по заголовку и т.п.
		Авторы: Артур Аюханов (artbear)
		
1.2 Компоненты, реализованные в виде DLL-файлов
	
	CommonServices.wsc
		Библиотека часто используемых в скриптах функций
		Авторы: Артур Аюханов, Алексей Диркс, Александр Кунташов
		
	Collections.wsc
		Реализует объект-коллекцию 
		Авторы: http://scrhostplugin.sourceforge.net/
		
	Registry.wsc
	OpenConf.RegistryIniFile.wsc
		Объекты для работы с реестром и ini-файлами
		Авторы: Артур Аюханов, Александр Кунташов
	
	1S.StatusIB.wsc
		Библиотека, рализующие функции определения статуса информационной базы 1С 
		в файл-серверном варианте (DBF)
		Авторы: Артур Аюханов
		
	SyntaxAnalysis.wsc
		Реалиует функции синтаксического анализа модулей 1С
		Авторы: Алексей Диркс
		
2. УСТАНОВКА И УДАЛЕНИЕ КОМПОНЕНТ

	Для того, чтобы иметь возможность создавать COM-объекты, реализуемые перечисленными 
компонентами, их требуется специальным образом зарегистрировать в системе. 

	Если установка скриптов для ОпенКонф производится при помощи автоматического 
инсталлятора, то компоненты будут зарегистрированы автоматически, в противном случае
(вы используете снапшот репозитария скриптов http://1c.alterplast.ru/files/ocs/)
Вам придется выполнить регистрацию компонент самостоятельно.
	В репозитарии системные файлы хранятся в подкаталоге "СистемныеФайлы".
Рекомендуется предварительно скопировать эти файлы в каталог
	<КаталогBinУстановки1С>\config\system
(создав при необходимости каталог system).

	Ниже описаны два способа регистрации компонент.
	
2.1 Регистрация компонент "вручную"

	Для регистрации компонент используется утилита regsvr32.exe, входящая
в поставку операционных систем семейства Windows.
	Чтобы зарегистрировать *.dll- или *.wsc-файл в системе, следует
запустить эту команду, передав в качестве параметра полный путь и имя регистрируемого
файла, например:

	regsvr32 C:\1Cv77\bin\config\system\SelectDialog.dll

При этом, если путь содержит пробелы, то весь путь следует взять в кавычки:

	regsvr32 "C:\Program Files\1Cv77\bin\config\system\CommonServices.wsc"

В случае успешной регистрации DLL-файла будет выдано сообщение 
	
	"DllRegisterServerEx in <Полный путь и имя файла> succeed."
	
а в случае WSC-файла

	"DllRegisterServerEx in <Путь к каталогу Windows>\system32\scrobj.dll succeed."

	Следует учитывать, что в общем случае регистрация компонент не может производиться
в произвольном порядке в виду того, что они могут использовать друг друга. В первую очередь
это касается компонент, реализованных в виде WSC - так называемых "скриптлетов".

	В настоящий момент регистрация компонент должна производится в следующем порядке:

	1.	В первую очередь регистрируются DLL-файлы (в произвольном порядке);
	2.	Затем регистрируется скриптлет CommonServices.wsc
	3.	Остальные скриптлеты регистрируются в произвольном порядке, за одним исключением:
		скриптлет Registry.wsc обязательно должен быть зарегистрирован раньше скриптлета
		OpenConf.RegistryIniFile.wsc
		
	Отмена регистрации компонент выполняется с помощью той же утилиты regsvr32.exe с 
использованием дополнительного параметра /U перед путем и именем файла:

	regsvr32 /U C:\1Cv77\bin\config\system\SelectDialog.dll

или

	regsvr32 /U "C:\Program Files\1Cv77\bin\config\system\CommonServices.wsc"

	Отмену регистрации компонент рекомендуется производить в порядке, обратном 
порядку их регистрации.
	
2.2 Использование скрипта regfiles.js

	В состав инсталлятора входит скрипт для автоматической регистрации компонент
служит скрипт regfiles.js. В снапшоте репозитария он располагается вместе с системными
файлами. 
	Скрипт выполняет последовательную регистрацию всех системных компонент, входящих 
в состав поставки в правильном порядке, а также умеет выполнять отмену регистрации.
	Для регистрации файлов скрипт следует запустить с параметром /I. Для отмены 
регистрации - с параметром /U. Подробнее о способах его запуска можно прочитать в 
комментарии скрипта, открыв его в блокноте.
	Можно также воспользоваться прилагаемыми bat-файлами: для регистрации всех системных
файлов следует перейти в директорию системных файлов (system) и запустить на исполнение
файл regall.bat. Для отмены регистрации - таким же образом запустить файл unregall.bat.
