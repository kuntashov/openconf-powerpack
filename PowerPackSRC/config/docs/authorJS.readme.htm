Версия

Скрипт: "Авторские комментарии"
Имя файла: author.js
Автор: Александр Кунташов kuntashov at [yandex.ru|gmail.com]

Зависимости

Для работы скрипта необходимо наличие установленных (зарегистрированных в системе) скриптлетов:

  • CommonServices.wsc?
  • Registry.wsc?

Описание

Скрипт для разметки кода по типу внесенных изменений (добавление, удаление, изменение) с добавлением сигнатуры (подписи) автора. (Такую разметку часто называют "следом").

Подпись можно гибко настраивать с использованием специальных переменных, которые позволяют формировать сигнатуру автоматически по заданному шаблону.

В шаблоне можно задать:

  • имя автора, в качестве которого возможно автоматическое использование
    • имени текущего пользователя 1С
    • логин пользователя операционной системы
    • произвольной строки
  • название организации
  • текущую дату и/или времени, формат представления которых задается пользователем

Все части подписи опциональны; дата форматируется в соответствии с заданным шаблоном. Настройка скрипта производится в диалоге.

Установка

Для работы скрипта необходим по крайней мере скриптлет CommonServices?.wsc последней версии. Для возможности настройки скрипта в диалоговорм режиме необходим также скриптлет Registry.wsc. Скорее всего оба они включены в поставку и находятся в архиве вместе с самим скриптом author.js.

Для установки скрипта:

  1. Cкопируйте его в папку
    <КаталогПрограммы()>\config\scripts\
  2. Перезапустите конфигуратор(ы).

(Предполагается, что названные выше скриптлеты у вас уже установлены.)

Для тех, кто не знает, как правильно установить скриптлеты:

  1. Оба скриптлета следует разместить в каталоге (если такого каталога не существует, его следует создать)
    <КаталогПрограммы()>\config\system\
  2. Скриптлеты следует зарегистрировать при помощи команды
    regsvr32 <имя_скриптлета>

Тем, кто знает :-)

Сравните установленные у вас скриптлеты с теми, что включены в поставку. Если в той версии, что установлена у вас нет некоторых функций (например, FindWindow?), то его надо обновить.

В поставку включены версии скриптлетов, последние на момент сборки данного "дистрибутива" или "пакета" - как угодно. Но не факт, что это самые последние версии на сегодня (я имею то время, когда вы читаете этот файл).

Самые последние версии скриптлетов, как и этого скрипта можно получить из репозитария на cvs.alterplast.ru. За инструкциями, как это сделать, отсылаю на http://1c.alterplast.ru.

Ежедневный снапшот репозитария можно скачать по ссылке

http://1c.alterplast.ru/files/ocs/ocs_last.zip

Последний способ настоятельно рекомендуется для тех, кто не знает, что такое CVS.

Макросы

МаркерДобавлен

Маркирует выделенный блок или текущую строку признаком "Добавлен"

МаркерИзменен

Копирует (опционально) выделенный блок или текущую строку, комментируя оригинал и маркирует блок/строку признаком "Изменен"

МаркерУдален

Комментирует выделенный блок/строку с одновременной установкой маркера "Удален"

ВставитьТекст

Вставляет текст из буфера обмена с добавлением соответствующих маркеров:

  • если перед выполнением макросов был выделен какой-либо блок, то он будет взят в качестве оригинального блока как при выполнении макроса МаркерИзменен, а текст из буфера будет взят в качестве измененного блока
  • если перед выполнением макроса курсор находится в пустой строке, то текст из буфера будет всавлен в текущую позицию с добавленим признака "Добавлен" как при выполнении макроса МаркерДобавлен

Настройка

Вызывает окно настройки параметров скрипта

Настройка

Все параметры скрипта можно настроить в диалоге, который вызывается с помощью запуска макроса Настройка.

Значения параметров настройки записываются в реестр Windows (для чтения и записи настроек используется скриптлет Registry.wsc?) в ветку

HKCU\Software\1C\1Cv7\7.7\OpenConf\Scripts\Авторские комментарии

Используются следующие параметры (все - строковые):

  • authorName - имя/никнейм автора, в строке распознаются переменные:

    • %1CUser% - текущий пользователь 1С
    • %OSUser% - текущий пользователь (операционной) системы

  • companyName - название организации (строкой)

  • dateFormat - формат даты, используется для формирования представления текущей даты/времени при выполнении подстановки спецпеременной :

    • yyyy - год (полностью, четыре цифры)
    • yy - год, только последние две цифры
    • mm - месяц (число; добавляется ведущий ноль, если mm<10)
    • dd - день месяца (число; добавляется ведущий ноль, если dd<10)
    • HH - часы
    • MM - минуты
    • SS - секунды

  • markerAdded - открывающий маркер добавленного блока

  • markerChanged - открывающий маркер измененного блока

  • markerDeleted - открывающий маркет удаленного блока

  • markerEndBlock - закрывающий маркет блока

  • oneliner - если задан, то используется в качестве маркера для однострочного комментария, если не задан, то в качестве маркера однострочного комментария будет использоваться соответствующий открывающий маркер блока

  • signature - сигнатура (подпись), которая подставляется после маркера. В сигнатуре выполняются подстановки для следующих переменных:

    • %Author% - заменяется на значение параметра authorName
    • %Company% - заменяется на значение параметра companyName
    • %Date% - заменяется на текущую дату, отформатированную в соответствии с dateFormat

  • splitter - разделитель кода при замене

  • doNotIndent - не использовать отступ при изменении/замене - как в "Замене кода" Артура Аюханова (по умолчанию используется отступ такой же, как в первой строке блока)

  • doNotSignAtEnd - не добавлять сигнатуру в закрывающий маркер блока

  • doNotCopyOldCode - не копировать старый код перед изменением
